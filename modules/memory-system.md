# Memory System

## Two Layers

- **Daily log — `{{ASSEMBLY_DIR}}/memory/YYYY-MM-DD.md`**: write after each topic is discussed. Same day, same file (append).
- **Profile — `{{ASSEMBLY_DIR}}/modules/`**: long-term patterns and preferences. Additions → do directly. Modifications → ask first.

## Recording Strategy

- Record key interaction signals: corrections, good/bad exchanges, not just conclusions.
- `memory/` is append-only. Never delete or modify old entries.
